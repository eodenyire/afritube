## 1️⃣ Overview

**Purpose:**
Provide a **central reference for all project documentation**, making it easy for developers, QA, DevOps, and stakeholders to navigate AfriTube’s full design, implementation, and progress reports.

**Audience:**

* Developers & Engineers
* Project Managers
* QA & Testers
* Investors & Stakeholders

**Structure:**
Documents are organized **chronologically from concept → design → implementation → progress tracking**.

## 3️⃣ Recommended Navigation

**Suggested workflow for team members:**

1. Start with **BRD** to understand business objectives and success metrics
2. Review **PRD** for product features and MVP scope
3. Study **SSDD** to understand technical design and system logic
4. Reference **Architecture** for system diagrams and scaling strategy
5. Check **Database Schema** for table structures, relationships, and security policies
6. Follow **Implementation Plan** for building & deploying the system
7. Monitor **Progress Reports** for current status, blockers, and next actions
